Design Issues:

Reusability - Each module will contain code that could be useful in another program. Examples: A* and distance calculations Maintainability - The code will adhere to strict style guidelines and will be heavily documented Testability - The style of development that we are employing, iterative in the REPL, means that we are testing while developing. Unit tests will also be used when possible and functions and functional units will be tested using unit tests. Performance - Performance was a consideration when choosing python since it is not known for its speed. Performance testing will be used on the most computationally intensive parts of the program. Most parts of the program will not be CPU bound but rather waiting for the user to interact. Portability - Python and the libraries we are using are extremely portable and we are targetting both Windows and Linux platforms. Mobile platforms (excluding iOS) would not be extraordinarily difficult to port to. Reuse - I already covered this. Safety - I don't know what you mean by this. Prototyping - Our development strategy is to build a prototype into a full product. I expect technical difficulties with the algorithm to match times with streets and the algorithm to find the correct route since these are both difficult problems. In choosing python we chose a working prototype iterative cycle with a drawback of everyone having to learn the language.
